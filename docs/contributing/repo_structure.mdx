# 저장소 구조

LangChain 코드나 문서에 기여할 계획이 있다면, 저장소의 높은 수준 구조를 이해하는 것이 유용할 수 있습니다.

LangChain은 여러 패키지를 포함하는 [모노레포](https://en.wikipedia.org/wiki/Monorepo)로 구성되어 있습니다. 이들이 어떻게 결합되는지에 대한 자세한 내용은 [설치 가이드](/docs/how_to/installation/)를 확인하세요.

다음은 트리 구조로 시각화한 구조입니다:

```text
.
├── cookbook # Tutorials and examples
├── docs # Contains content for the documentation here: https://python.langchain.com/
├── libs
│   ├── langchain
│   │   ├── langchain
│   │   ├── tests/unit_tests # Unit tests (present in each package not shown for brevity)
│   │   ├── tests/integration_tests # Integration tests (present in each package not shown for brevity)
│   ├── community # Third-party integrations
│   │   ├── langchain-community
│   ├── core # Base interfaces for key abstractions
│   │   ├── langchain-core
│   ├── experimental # Experimental components and chains
│   │   ├── langchain-experimental
|   ├── cli # Command line interface
│   │   ├── langchain-cli
│   ├── text-splitters
│   │   ├── langchain-text-splitters
│   ├── standard-tests
│   │   ├── langchain-standard-tests
│   ├── partners
│       ├── langchain-partner-1
│       ├── langchain-partner-2
│       ├── ...
│
├── templates # A collection of easily deployable reference architectures for a wide variety of tasks.
```


루트 디렉토리에는 다음 파일도 포함되어 있습니다:

* `pyproject.toml`: 문서 작성 및 린트, 요리책을 위한 의존성.
* `Makefile`: 빌드, 린트 및 문서와 요리책을 위한 단축키가 포함된 파일.

루트 디렉토리 수준에는 다른 파일도 있지만, 그 존재는 자명해야 합니다. 자유롭게 둘러보세요!

## 문서

`/docs` 디렉토리에는 https://python.langchain.com/ 에 표시되는 문서 내용과 관련 API 참조 https://api.python.langchain.com/en/latest/langchain_api_reference.html 가 포함되어 있습니다.

문서에 기여하는 방법을 배우려면 [문서](/docs/contributing/documentation/) 가이드라인을 참조하세요.

## 코드

`/libs` 디렉토리에는 LangChain 패키지의 코드가 포함되어 있습니다.

코드 기여 방법에 대한 자세한 내용은 다음 가이드라인을 참조하세요:

- [코드](/docs/contributing/code/): LangChain 코드베이스에서 개발하는 방법을 배우세요.
- [통합](./integrations.mdx): `langchain-community`에 대한 서드파티 통합에 기여하거나 새로운 파트너 패키지를 시작하는 방법을 배우세요.
- [테스트](./testing.mdx): 패키지에 대한 테스트를 작성하는 방법을 배우는 가이드라인.
