# 설정

LangChain 문서는 두 가지 구성 요소로 이루어져 있습니다:

1. 주요 문서: [python.langchain.com](https://python.langchain.com/)에서 호스팅되며,
이 포괄적인 리소스는 주요 사용자 문서로 사용됩니다.
튜토리얼, 사용 사례, 통합 등 다양한 주제를 다루며,
LangChain으로 구축하는 데 대한 광범위한 지침을 제공합니다.
이 문서의 내용은 모노레포의 `/docs` 디렉토리에 있습니다.
2. 코드 내 문서: 이는 코드베이스 자체에 대한 문서로,
외부에서 볼 수 있는 [API 참조](https://api.python.langchain.com/en/latest/langchain_api_reference.html)를 생성하는 데에도 사용됩니다.
API 참조의 내용은 코드베이스의 docstring을 스캔하여 자동 생성됩니다. 이러한 이유로 우리는
개발자들이 코드를 잘 문서화할 것을 요청합니다.

`API 참조`는 주로 [sphinx](https://www.sphinx-doc.org/en/master/)에 의해 코드에서 자동 생성되며,
[Read the Docs](https://readthedocs.org/)에 호스팅됩니다.

우리는 문서에 대한 모든 기여를 감사히 여기며, 오타 수정,
새로운 튜토리얼이나 예제 추가, 주요 문서 또는 API 참조에 관계없이 기여를 환영합니다.

린팅과 유사하게, 우리는 문서화가 번거로울 수 있음을 인식합니다. 만약 당신이 이를 원하지 않는다면,
프로젝트 유지 관리자에게 연락해 주시면 도움을 드릴 수 있습니다. 우리는 이것이 좋은 코드 기여의 장애물이 되기를 원하지 않습니다.

## 📜 주요 문서

주요 문서의 내용은 모노레포의 `/docs` 디렉토리에 있습니다.

문서는 ipython 노트북(`.ipynb` 파일)과 마크다운(`.mdx` 파일)의 조합을 사용하여 작성됩니다.
노트북은 마크다운으로 변환된 후 [Docusaurus 2](https://docusaurus.io/)를 사용하여 빌드됩니다.

주요 문서에 기여해 주셔도 좋습니다! 🥰

문서를 수정한 후:

1. 문서가 잘 포맷되고 오류가 없는지 확인하기 위해 린팅 및 포맷팅 명령을 실행합니다(아래 참조).
2. 선택적으로 문서를 로컬에서 빌드하여 변경 사항이 잘 보이는지 확인합니다.
3. 변경 사항을 포함한 풀 리퀘스트를 만듭니다.
4. 풀 리퀘스트 `대화` 페이지에서 `배포 보기` 또는 `미리보기 방문` 버튼을 클릭하여 변경 사항이 원하는 대로 되었는지 미리 보고 확인할 수 있습니다. 이는 문서 변경 사항의 미리보기를 보여줍니다.

## ⚒️ 로컬에서 문서 린팅 및 빌드

문서를 작성한 후, 문서가 잘 보이고 오류가 없는지 확인하기 위해 로컬에서 린팅 및 빌드를 원할 수 있습니다.

로컬에서 빌드할 수 없다면 괜찮습니다. 풀 리퀘스트 페이지에서 문서의 미리보기를 볼 수 있습니다.

**모노레포 루트**에서 다음 명령을 실행하여 종속성을 설치합니다:

```bash
poetry install --with lint,docs --no-root
```


### 빌드

문서를 빌드하는 코드는 모노레포의 `/docs` 디렉토리에 있습니다.

다음 명령에서 접두사 `api_`는 API 참조를 위한 작업임을 나타냅니다.

문서를 빌드하기 전에 빌드 디렉토리를 정리하는 것이 항상 좋은 아이디어입니다:

```bash
make docs_clean
make api_docs_clean
```


다음으로, 아래에 설명된 대로 문서를 빌드할 수 있습니다:

```bash
make docs_build
make api_docs_build
```


:::tip

`make api_docs_build` 명령은 시간이 오래 걸립니다. API 문서에 대한 미관상의 변경을 하고 어떻게 보이는지 확인하고 싶다면, 다음을 사용하세요:

```bash
make api_docs_quick_preview
```


이는 API 참조의 작은 하위 집합만 빌드할 것입니다.

:::

마지막으로 모든 링크가 유효한지 확인하기 위해 링크 체커를 실행합니다:

```bash
make docs_linkcheck
make api_docs_linkcheck
```


### 린팅 및 포맷팅

주요 문서는 **모노레포 루트**에서 린팅됩니다. 주요 문서를 린팅하려면, 거기에서 다음을 실행합니다:

```bash
make lint
```


포맷팅 관련 오류가 있는 경우, 다음을 사용하여 자동으로 수정할 수 있습니다:

```bash
make format
```


## ⌨️ 코드 내 문서

코드 내 문서는 주로 [sphinx](https://www.sphinx-doc.org/en/master/)에 의해 코드에서 자동 생성되며,
[Read the Docs](https://readthedocs.org/)에 호스팅됩니다.

API 참조가 유용하려면 코드베이스가 잘 문서화되어야 합니다. 이는 모든 함수, 클래스 및 메서드가 무엇을 하는지, 인수가 무엇인지, 반환 값이 무엇인지 설명하는 docstring을 가져야 함을 의미합니다. 이는 일반적으로 좋은 관행이지만, LangChain에 특히 중요합니다. 왜냐하면 API 참조가 개발자가 코드베이스를 사용하는 방법을 이해하는 데 주요 리소스이기 때문입니다.

우리는 일반적으로 docstring에 대해 [Google Python Style Guide](https://google.github.io/styleguide/pyguide.html#38-comments-and-docstrings)를 따릅니다.

다음은 잘 문서화된 함수의 예입니다:

```python

def my_function(arg1: int, arg2: str) -> float:
    """This is a short description of the function. (It should be a single sentence.)

    This is a longer description of the function. It should explain what
    the function does, what the arguments are, and what the return value is.
    It should wrap at 88 characters.

    Examples:
        This is a section for examples of how to use the function.

        .. code-block:: python

            my_function(1, "hello")

    Args:
        arg1: This is a description of arg1. We do not need to specify the type since
            it is already specified in the function signature.
        arg2: This is a description of arg2.

    Returns:
        This is a description of the return value.
    """
    return 3.14
```


### 린팅 및 포맷팅

코드 내 문서는 문서화되는 패키지에 속한 디렉토리에서 린팅됩니다.

예를 들어, `langchain-community` 패키지에서 작업 중이라면,
작업 디렉토리를 `langchain-community` 디렉토리로 변경해야 합니다:

```bash
cd [root]/libs/langchain-community
```


패키지를 위한 가상 환경을 설정합니다. 아직 설정하지 않았다면.

패키지의 종속성을 설치합니다.

```bash
poetry install --with lint
```


그런 다음 다음 명령을 실행하여 코드 내 문서를 린팅하고 포맷할 수 있습니다:

```bash
make format
make lint
```


## 문서 변경 사항 확인

문서 변경 사항을 리포지토리에 푸시한 후, `배포 보기` 또는 `미리보기 방문` 버튼을 클릭하여 변경 사항이 원하는 대로 되었는지 미리 보고 확인할 수 있습니다.
이는 문서 변경 사항의 미리보기를 보여줍니다.
이 미리보기는 [Vercel](https://vercel.com/docs/getting-started-with-vercel)에 의해 생성됩니다.
