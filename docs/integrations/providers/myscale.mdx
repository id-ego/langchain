# MyScale

이 페이지는 LangChain 내에서 MyScale 벡터 데이터베이스를 사용하는 방법을 다룹니다. 설치 및 설정과 특정 MyScale 래퍼에 대한 참조로 나누어져 있습니다.

MyScale을 사용하면 구조화된 데이터와 비구조화된(벡터화된) 데이터를 모두 관리하고, SQL을 사용하여 두 유형의 데이터에 대한 공동 쿼리 및 분석을 수행할 수 있습니다. 또한, ClickHouse 위에 구축된 MyScale의 클라우드 네이티브 OLAP 아키텍처는 대규모 데이터 세트에서도 번개처럼 빠른 데이터 처리를 가능하게 합니다.

## 소개

[MyScale 및 고성능 벡터 검색 개요](https://docs.myscale.com/en/overview/)

이제 우리의 SaaS에 등록하고 [클러스터를 시작하세요!](https://docs.myscale.com/en/quickstart/)

SQL과 벡터를 통합하는 방법에 관심이 있으시다면, 추가 구문 참조를 위해 [이 문서](https://docs.myscale.com/en/vector-reference/)를 참조하시기 바랍니다.

우리는 또한 huggingface에서 라이브 데모를 제공합니다! 우리의 [huggingface 공간](https://huggingface.co/myscale)을 확인해 보세요! 그들은 눈 깜짝할 사이에 수백만 개의 벡터를 검색합니다!

## 설치 및 설정
- `pip install clickhouse-connect`로 Python SDK를 설치하세요.

### 환경 설정

myscale 인덱스의 매개변수를 설정하는 방법은 두 가지가 있습니다.

1. 환경 변수
   
   앱을 실행하기 전에 `export`로 환경 변수를 설정하세요:
`export MYSCALE_HOST='<your-endpoints-url>' MYSCALE_PORT=<your-endpoints-port> MYSCALE_USERNAME=<your-username> MYSCALE_PASSWORD=<your-password> ...`
   
   귀하의 계정, 비밀번호 및 기타 정보는 우리의 SaaS에서 쉽게 찾을 수 있습니다. 자세한 내용은 [이 문서](https://docs.myscale.com/en/cluster-management/)를 참조하시기 바랍니다. `MyScaleSettings` 아래의 모든 속성은 접두사 `MYSCALE_`로 설정할 수 있으며 대소문자를 구분하지 않습니다.
2. 매개변수로 `MyScaleSettings` 객체 생성
   
   ```python
   from langchain_community.vectorstores import MyScale, MyScaleSettings
   config = MyScaleSettings(host="<your-backend-url>", port=8443, ...)
   index = MyScale(embedding_function, config)
   index.add_documents(...)
   ```
