# 비구조적

> `unstructured` 패키지는 [Unstructured.IO](https://www.unstructured.io/)에서 제공하며, PDF 및 Word 문서와 같은 원시 소스 문서에서 깨끗한 텍스트를 추출합니다. 이 페이지에서는 LangChain 내에서 [`unstructured`](https://github.com/Unstructured-IO/unstructured) 생태계를 사용하는 방법을 다룹니다.

## 설치 및 설정

로컬에서 실행되는 로더를 사용하고 있다면, `unstructured` 및 그 의존성을 실행하기 위해 다음 단계를 따르세요.

- 가장 작은 설치 용량을 위해, 오픈 소스 `unstructured` 패키지에서 사용할 수 없는 기능을 활용하려면, `pip install unstructured-client`로 Python SDK를 설치하고 `pip install langchain-unstructured`를 함께 설치하여 `UnstructuredLoader`를 사용하고 Unstructured API에 대해 원격으로 파티셔닝하세요. 이 로더는 `langchain-community` 레포가 아닌 LangChain 파트너 레포에 있으며, 무료 키를 생성할 수 있는 `api_key`가 필요합니다 [여기](https://unstructured.io/api-key/)에서 생성할 수 있습니다.
  - SDK에 대한 Unstructured의 문서는 여기에서 확인할 수 있습니다: https://docs.unstructured.io/api-reference/api-services/sdk
- 모든 것을 로컬에서 실행하려면, 오픈 소스 Python 패키지를 `pip install unstructured`로 설치하고 위에서 언급한 것과 동일한 `UnstructuredLoader`를 사용하세요.
  - 문서 특정 의존성은 extras로 설치할 수 있습니다, 예: `pip install "unstructured[docx]"`.
  - 모든 문서 유형에 대한 의존성을 설치하려면 `pip install "unstructured[all-docs]"`를 사용하세요.
- 시스템에 아직 설치되지 않은 경우, 다음 시스템 의존성을 설치하세요. Mac의 경우 `brew install`을 사용할 수 있습니다. 어떤 문서 유형을 파싱하는지에 따라 이 모든 것이 필요하지 않을 수 있습니다.
  - `libmagic-dev` (파일 유형 감지)
  - `poppler-utils` (이미지 및 PDF)
  - `tesseract-ocr` (이미지 및 PDF)
  - `qpdf` (PDF)
  - `libreoffice` (MS Office 문서)
  - `pandoc` (EPUB)
- 로컬에서 실행할 때, Unstructured는 모든 시스템 의존성이 올바르게 설치되도록 [이 가이드](https://docs.unstructured.io/open-source/installation/docker-installation)를 따라 Docker를 사용하는 것을 권장합니다.

Unstructured API는 요청을 하기 위해 API 키가 필요합니다. [여기](https://unstructured.io/api-key-hosted)에서 API 키를 요청하고 오늘부터 사용을 시작하세요! API 호출을 시작하는 방법은 [여기](https://github.com/Unstructured-IO/unstructured-api)에서 README를 확인하세요. 여러분의 피드백을 듣고 싶습니다, [커뮤니티 슬랙](https://join.slack.com/t/unstructuredw-kbe4326/shared_invite/zt-1x7cgo0pg-PTptXWylzPQF9xZolzCnwQ)에서 어떻게 진행되는지 알려주세요. 품질과 성능 개선을 기대해 주세요! Unstructured API를 자체 호스팅하거나 로컬에서 실행하고 싶다면 [여기](https://github.com/Unstructured-IO/unstructured-api#dizzy-instructions-for-using-the-docker-image)에서 지침을 확인하세요.
