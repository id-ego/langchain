## 채팅 모델
### Azure OpenAI

> [Microsoft Azure](https://en.wikipedia.org/wiki/Microsoft_Azure), 일반적으로 `Azure`라고 불리는 이 플랫폼은 `Microsoft`가 운영하는 클라우드 컴퓨팅 플랫폼으로, 글로벌 데이터 센터를 통해 애플리케이션 및 서비스에 대한 접근, 관리 및 개발을 제공합니다. ### Azure Blob Storage

> [Azure Blob Storage](https://learn.microsoft.com/en-us/azure/storage/blobs/storage-blobs-introduction)는 Microsoft의 클라우드 객체 저장 솔루션입니다. Blob Storage는 대량의 비구조적 데이터를 저장하는 데 최적화되어 있습니다. 비구조적 데이터는 특정 데이터 모델이나 정의를 따르지 않는 데이터로, 텍스트나 이진 데이터와 같은 형태입니다.

> [Azure Files](https://learn.microsoft.com/en-us/azure/storage/files/storage-files-introduction)는 업계 표준 서버 메시지 블록(`SMB`) 프로토콜, 네트워크 파일 시스템(`NFS`) 프로토콜 및 `Azure Files REST API`를 통해 접근할 수 있는 완전 관리형 파일 공유를 클라우드에서 제공합니다. `Azure Files`는 `Azure Blob Storage`를 기반으로 합니다.

`Azure Blob Storage`는 다음과 같은 용도로 설계되었습니다:
- 이미지를 브라우저에 직접 제공.
- 분산 액세스를 위한 파일 저장.
- 비디오 및 오디오 스트리밍.
- 로그 파일 작성.
- 백업 및 복원, 재해 복구 및 아카이빙을 위한 데이터 저장.
- 온프레미스 또는 Azure 호스팅 서비스에 의한 데이터 분석을 위한 데이터 저장.

```bash
pip install azure-storage-blob
```

## AI 에이전트 메모리 시스템

[AI 에이전트](https://learn.microsoft.com/en-us/azure/cosmos-db/ai-agents)는 다중 모드를 지원하고 강력한 운영 성능을 제공하며 에이전트 메모리 공유 및 분리를 가능하게 하는 강력한 메모리 시스템이 필요합니다.

AI 에이전트는 Azure Cosmos DB를 통합된 [메모리 시스템](https://learn.microsoft.com/en-us/azure/cosmos-db/ai-agents#memory-can-make-or-break-agents) 솔루션으로 활용할 수 있으며, 속도, 확장성 및 단순성을 누릴 수 있습니다. 이 서비스는 OpenAI의 ChatGPT 서비스를 [성공적으로 지원](https://www.youtube.com/watch?v=6IIUtEFKJec&t)하여 높은 신뢰성과 낮은 유지 관리로 동적으로 확장할 수 있게 합니다. 원자 기록 시퀀스 엔진에 의해 구동되며, 이는 세계 최초의 전 세계적으로 분산된 [NoSQL](https://learn.microsoft.com/en-us/azure/cosmos-db/distributed-nosql), [관계형](https://learn.microsoft.com/en-us/azure/cosmos-db/distributed-relational) 및 [벡터 데이터베이스](https://learn.microsoft.com/en-us/azure/cosmos-db/vector-database) 서비스로 서버리스 모드를 제공합니다.

아래는 벡터 저장 기능을 제공할 수 있는 두 가지 Azure Cosmos DB API입니다.

### Azure Cosmos DB for MongoDB (vCore)

> [Azure Cosmos DB for MongoDB vCore](https://learn.microsoft.com/en-us/azure/cosmos-db/mongodb/vcore/)는 전체 네이티브 MongoDB 지원으로 데이터베이스를 쉽게 생성할 수 있게 합니다.
MongoDB 경험을 적용하고 애플리케이션을 MongoDB vCore 계정의 연결 문자열에 지정하여 좋아하는 MongoDB 드라이버, SDK 및 도구를 계속 사용할 수 있습니다.
Azure Cosmos DB for MongoDB vCore에서 벡터 검색을 사용하여 AI 기반 애플리케이션과 Azure Cosmos DB에 저장된 데이터를 원활하게 통합할 수 있습니다.

### Azure Cosmos DB NoSQL

> [Azure Cosmos DB for NoSQL](https://learn.microsoft.com/en-us/azure/cosmos-db/nosql/vector-search)는 이제 벡터 인덱싱 및 검색을 미리 보기로 제공합니다.
이 기능은 고차원 벡터를 처리하도록 설계되어 있으며, 모든 규모에서 효율적이고 정확한 벡터 검색을 가능하게 합니다. 이제 벡터를 데이터와 함께 문서에 직접 저장할 수 있습니다. 즉, 데이터베이스의 각 문서는 전통적인 스키마 없는 데이터뿐만 아니라 문서의 다른 속성으로서 고차원 벡터를 포함할 수 있습니다. 데이터와 벡터의 이 같은 동시 배치는 효율적인 인덱싱 및 검색을 가능하게 하며, 벡터는 표현하는 데이터와 동일한 논리적 단위에 저장됩니다. 이는 데이터 관리, AI 애플리케이션 아키텍처 및 벡터 기반 작업의 효율성을 단순화합니다.

## 검색기
### Azure AI 검색

> [Azure AI Search](https://learn.microsoft.com/en-us/azure/search/search-what-is-azure-search) (이전의 `Azure Search` 또는 `Azure Cognitive Search`)는 클라우드 검색 서비스로, 개발자에게 웹, 모바일 및 기업 애플리케이션에서 개인의 이질적인 콘텐츠에 대한 풍부한 검색 경험을 구축하기 위한 인프라, API 및 도구를 제공합니다.

> 검색은 사용자에게 텍스트를 제공하는 모든 앱의 기초가 되며, 일반적인 시나리오는 카탈로그 또는 문서 검색, 온라인 소매 앱 또는 독점 콘텐츠에 대한 데이터 탐색을 포함합니다. 검색 서비스를 생성할 때 다음 기능을 사용하게 됩니다:
> - 사용자 소유 콘텐츠가 포함된 검색 인덱스에 대한 전체 텍스트 검색을 위한 검색 엔진
> - 콘텐츠 추출 및 변환을 위한 선택적 AI 강화와 함께 풍부한 인덱싱, 어휘 분석 포함
> - 텍스트 검색, 퍼지 검색, 자동 완성, 지리 검색 등을 위한 풍부한 쿼리 구문
> - Azure SDK의 REST API 및 클라이언트 라이브러리를 통한 프로그래머블성
> - 데이터 계층, 기계 학습 계층 및 AI(AI 서비스)에서의 Azure 통합

[설정 지침](https://learn.microsoft.com/en-us/azure/search/search-create-service-portal)을 참조하세요.
