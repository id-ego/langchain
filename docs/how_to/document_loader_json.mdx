# JSON 로드하는 방법

[JSON (JavaScript Object Notation)](https://en.wikipedia.org/wiki/JSON)은 속성-값 쌍과 배열(또는 기타 직렬화 가능한 값)으로 구성된 데이터 객체를 저장하고 전송하기 위해 사람이 읽을 수 있는 텍스트를 사용하는 개방형 표준 파일 형식 및 데이터 교환 형식입니다.

[JSON Lines](https://jsonlines.org/)는 각 줄이 유효한 JSON 값인 파일 형식입니다.

LangChain은 JSON 및 JSONL 데이터를 LangChain [Document](https://api.python.langchain.com/en/latest/documents/langchain_core.documents.base.Document.html#langchain_core.documents.base.Document) 객체로 변환하기 위해 [JSONLoader](https://api.python.langchain.com/en/latest/document_loaders/langchain_community.document_loaders.json_loader.JSONLoader.html)를 구현합니다. 이는 지정된 [jq 스키마](https://en.wikipedia.org/wiki/Jq_(programming_language))를 사용하여 JSON 파일을 구문 분석하고, 특정 필드를 LangChain Document의 내용 및 메타데이터로 추출할 수 있도록 합니다.

이는 `jq` 파이썬 패키지를 사용합니다. `jq` 구문에 대한 자세한 문서는 이 [매뉴얼](https://stedolan.github.io/jq/manual/#Basicfilters)을 참조하세요.

여기서는 다음을 시연합니다:

- LangChain `Document`의 내용으로 JSON 및 JSONL 데이터를 로드하는 방법;
- `Document`와 관련된 메타데이터로 JSON 및 JSONL 데이터를 로드하는 방법.

    jq_schema='.messages[].content',

data = loader.load()

## 메타데이터 추출

일반적으로 우리는 JSON 파일에서 사용할 수 있는 메타데이터를 우리가 생성하는 문서에 포함하고자 합니다.

다음은 `JSONLoader`를 사용하여 메타데이터를 추출하는 방법을 보여줍니다.

주목할 만한 몇 가지 주요 변경 사항이 있습니다. 메타데이터를 수집하지 않았던 이전 예제에서는 `page_content`의 값을 추출할 수 있는 위치를 스키마에서 직접 지정할 수 있었습니다.

```
.messages[].content
```


현재 예제에서는 로더에게 `messages` 필드의 레코드를 반복하도록 지시해야 합니다. 그러면 jq_schema는 다음과 같아야 합니다:

```
.messages[]
```


이렇게 하면 레코드(dict)를 최종 `Document` 객체에 저장된 메타데이터에 포함해야 하는 정보를 식별하는 `metadata_func`로 전달할 수 있습니다.

또한 이제 로더에서 `content_key` 인수를 통해 레코드에서 `page_content`의 값을 추출해야 하는 키를 명시적으로 지정해야 합니다.

```python
# Define the metadata extraction function.
def metadata_func(record: dict, metadata: dict) -> dict:

    metadata["sender_name"] = record.get("sender_name")
    metadata["timestamp_ms"] = record.get("timestamp_ms")

    return metadata


loader = JSONLoader(
    file_path='./example_data/facebook_chat.json',
    jq_schema='.messages[]',
    content_key="content",
    metadata_func=metadata_func
)

data = loader.load()
```

## `metadata_func`

위에서 보았듯이, `metadata_func`는 `JSONLoader`에 의해 생성된 기본 메타데이터를 수락합니다. 이는 메타데이터 형식에 대해 사용자에게 완전한 제어를 허용합니다.

예를 들어, 기본 메타데이터에는 `source` 및 `seq_num` 키가 포함됩니다. 그러나 JSON 데이터에도 이러한 키가 포함될 수 있습니다. 사용자는 `metadata_func`를 활용하여 기본 키의 이름을 바꾸고 JSON 데이터의 키를 사용할 수 있습니다.

아래 예제는 `source`를 수정하여 `langchain` 디렉토리에 상대적인 파일 소스 정보만 포함하도록 하는 방법을 보여줍니다.

```python
# Define the metadata extraction function.
def metadata_func(record: dict, metadata: dict) -> dict:

    metadata["sender_name"] = record.get("sender_name")
    metadata["timestamp_ms"] = record.get("timestamp_ms")

    if "source" in metadata:
        source = metadata["source"].split("/")
        source = source[source.index("langchain"):]
        metadata["source"] = "/".join(source)

    return metadata


loader = JSONLoader(
    file_path='./example_data/facebook_chat.json',
    jq_schema='.messages[]',
    content_key="content",
    metadata_func=metadata_func
)

data = loader.load()
```

## jq 스키마가 있는 일반적인 JSON 구조

아래 목록은 구조에 따라 JSON 데이터에서 내용을 추출하기 위해 사용자가 사용할 수 있는 가능한 `jq_schema`에 대한 참조를 제공합니다.

```
JSON        -> [{"text": ...}, {"text": ...}, {"text": ...}]
jq_schema   -> ".[].text"

JSON        -> {"key": [{"text": ...}, {"text": ...}, {"text": ...}]}
jq_schema   -> ".key[].text"

JSON        -> ["...", "...", "..."]
jq_schema   -> ".[]"
```
